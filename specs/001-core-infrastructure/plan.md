## Summary

Establish the full foundational infrastructure for the Moto Orbito Flutter app so every
Phase 1 feature module can be built against a stable, tested, and enforced set of core
primitives. The plan covers 12 subsystems delivered in dependency order:

1. Error types (`ApiResult<T>`, `Failure` hierarchy)
2. App-level DI bootstrap
3. Theme system (light + dark, `AppColorsExtension`)
4. Localization (`slang`, Arabic primary, strict parity enforcement)
5. Logging (`AppLogger`, flavor-gated)
6. Networking (`BaseApiClient` via Dio)
7. Supabase service layer (DB + Realtime + Storage)
8. FCM service (token lifecycle, foreground display, nav on tap)
9. AI service (Edge Function proxy, retry, prompt constants)
10. GoRouter (named routes, auth guard, deep link intent)
11. Shared widgets (RTL-aware, theme-aware, fully localized)
12. App shell (entry point wiring, bottom nav, no-connection screen)

All subsystems are independently testable. No Phase 1 feature code is scaffolded here.

## Technical Context

**Language/Version**: Dart 3.12 / Flutter 3.32+

**Primary Dependencies** (all pre-approved in `AGENTS.md §18`):
- `flutter_bloc` — Cubit/Bloc state management
- `get_it` — dependency injection
- `go_router` — named-route navigation + deep links
- `dio` — HTTP client
- `supabase_flutter` — BaaS (auth, DB, realtime, storage)
- `firebase_core` + `firebase_messaging` — FCM
- `flutter_local_notifications` — foreground push display
- `flutter_secure_storage` — JWT + FCM token persistence
- `slang` — localization (strict parity mode)
- `intl` — locale-aware date/number formatting
- `flutter_screenutil` — responsive sizing
- `equatable` — entity comparison
- `connectivity_plus` — network state detection
- `cached_network_image` — image loading
- `fl_chart` — dashboard charts (registered but unused in Phase 0)
- `mocktail` — test mocking

**Storage**: Supabase Postgres (remote); `flutter_secure_storage` (local secure)

**Testing**: `flutter_test` + `mocktail`

**Target Platform**: Android (primary), iOS

**Project Type**: Mobile app (Flutter)

**Performance Goals**:
- App launch < 3s on mid-range Android (development flavor)
- Language switch < 500ms, no visual flash

**Constraints**:
- No hardcoded strings, colors, or sizes anywhere in `lib/core/`
- No `AppConfig`, `--dart-define`, or `.env` — flavor config lives in entry points only
- `kDebugMode`/`kReleaseMode` MUST NOT be used as flavor proxy
- Phase 2/3 modules: zero scaffolding or references

**Scale/Scope**: Phase 0 — 12 core subsystems, ~60 files, ~40 tests
